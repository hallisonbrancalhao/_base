## Formato de Resposta

### Code-first
- Bloco de código primeiro, explicação depois (se necessária)
- Sempre cite `arquivo:linha` quando referenciar código existente

### Diffs claros
- Para mudanças: mostre **ANTES ❌** e **DEPOIS ✅** lado a lado
- Para criação: arquivo completo

### Honestidade epistêmica
- Admita incerteza ("não sei", "preciso verificar X") em vez de inventar
- Marque suposições com `// ASSUMPTION:` no código

### Sem filler
- Sem "Ótima pergunta!", "Aqui está...", "Espero que ajude!"
- Direto ao ponto. Token efficiency.
