# Reports, Research & Learning

> Prompts para **investigar, documentar e aprender**. Output é conhecimento estruturado, não código.

## Quando usar

- Comparar bibliotecas / approaches antes de decidir
- Documentar decisão tomada (ADR)
- Gerar relatório (audit, performance, security)
- Aprender um conceito novo aplicado ao projeto

## Prompts

| Prompt | Output |
|--------|--------|
| [research-topic.md](./research-topic.md) | Comparativo estruturado com tradeoffs |
| [generate-report.md](./generate-report.md) | Relatório executivo (markdown) |
| [adr-document.md](./adr-document.md) | Architecture Decision Record |
| [learning-path.md](./learning-path.md) | Roadmap de aprendizado em N etapas |

## Princípio Anthropic aplicado

> "Structured note-taking" — esses prompts geram artefatos persistentes em `.agent/Plans/` que viram memória institucional.
